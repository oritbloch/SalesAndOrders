משימת בית - מידע על מכירות מוצרים

מצורפים 2 פרוייקטים וקבצי סקריפט עבור SP.

קבצי הסקריפט מכילים יצירת SP בשם: SP_GetNumOfSalesAndOrdersData ו SP_GetOrderTimesAndDates.

SalesDataServices - פרוייקט עבור API בצד שרת לצורך חשיפת מידע מהSP. הגדרתי את הconnectionstring בקובץ הקונפיג.

GetSalesInformationClient - פרוייקט הקליינט - האתר. מתממשק במידת הצורך לפרוייקט השרת והלינק אליו מוגדר בקובץ קונפיג.


להלן הסבר קצר על התממשקות חלקי המשימה בתוך הפרוייקטים:

1. הצגת טבלה עבור כמות מכירות המוצרים לפי עיר ומוצר:

מצד שרת - יש קונטרולר בשם GetNumOfSales שצורך על ידי הSERVICE הייעודי את הנתונים שמובאים מה SP_GetNumOfSalesAndOrdersData.

מצד הקליינט - יש רכיב על הדף בשם SalesNumber שקורא למתודה GetProductSales שמביאה נתונים מה לAPI בצד שרת.

2. הצגת טבלה עבור הזמנות שסופקו לפני/אחרי הזמן:

מצד שרת - יש קונטרולר בשם GetOrderTimes שצורך על ידי הSERVICE הייעודי את הנתונים שמובאים מה SP_GetOrderTimesAndDates.

מצד הקליינט - יש רכיב על הדף בשם OrderTimes שקורא למתודה GetOrderTimes שמביאה נתונים מה לAPI בצד שרת.

3. טיימר בראש הדף מופיע על ידי הרכיב Timer והוא משתמש בlocalstorage לשמירה רציפה על כמות הזמן. לאיפוס המידע הוספתי כפתור Stop Counting.

4. על מנת להציג הודעה מצד שרת לקליינט השתמשתי ברכיב SignalR. לצורך כך יש להתקין אותו מצד הפרוייקט ומצד הטרמינל. השתמשתי בפקודות הבאות:

Install-Package Microsoft.AspNetCore.SignalR

npm install @microsoft/signalr

יש גירסאות של .NET CORE שבהן יעבוד הרכיב הזה במקום הקודם: npm install @aspnet/signalr.

מצד שרת הקמתי את class TimerHub, הוספתי הגדרות מתאימות בprogram.cs.

מצד האתר עצמו הוספתי רכיב לדף שנקרא GetMessageByTimer והוא יוצר connection עם הHUB שהוגדר בשרת ולהציג את ההודעה על הדף כשהיא מגיעה.
